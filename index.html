Spatial Analysis and Recording Tool (SPARTa)

SPARTa is the product from a year-long senior design capstone project as a requirement for graduation from Penn State's Software Engineering program. It included a collaboration between Penn State and Erie Insurance to utilize Google's Project Tango to improve the insurance claims process. The system can spatially analyze and scan a claim, being able to capture measurements, images, and full 3D models of rooms, buildings, or automobiles. This requires less physical measurements from adjusters helping to automate the process.

The project followed multiple stages in the software process such as requirement gathering, design, development, and testing. The project involved working with experimental technology, testing in the field, and communicating with various engineers and analysts from Erie Insurance, Penn State, and Google.


To Become a Henchling


To Become a Henchling is a 2D top-down, rouge-like about a little girl and her quest to do evil. The game was developed using Unity and features cartoon-like vector art assets, lovable characters, linear story, and bears! The game is a part of the capstone for Penn State Behrend's game development minor program.

Battleship

As a part of the final project for Penn State's Software Architecture (SWENG421), Battleship is a recreation of the classic Hasbro's BattleShip. Using Windows Forms and C#, Battleship is a fully playable version of the game utilizing basic Windows Forms graphics. The game featured two game modes, Three Ship and Five Ship. Five Ship is the full game while Three Ship is a shortened version with the two larger ships removed. Players share a single screen and take turns placing their pieces and then firing at the enemy ships.

It's also possible to save the game. From the start screen, specific bin files can be chosen that will reload the game to the previous working state. The game utilizes many software patterns, including factory, interfaces, abstract classes, memento/snapshot, singleton.



iCivilization

iCivilization is a city simulation game that was built using OpenGL, C++, object oriented practices. The game let's the player construct their city from the ground up from a variety of buildings: mills, houses, farms, and more. They must manage their resources and handle random events that will grant benefits or spell disaster.

The program uses OpenGL to create the all basic shapes, apply textures, and traverse objects in 3D space. This project was a part of Penn State's Computer Graphics course (CMPSC457) and Advanced Game Programming (GAME450).


File Locker Storage System (FLoSS)

FLoSS is a file management system developed using Java and GUI elements. In addition, the application was designed using object-oriented practices and UML diagrams. The client-server application allows for multiple users to store files, profile info, and contacts within a database. FLoSS allows users to register an account, upload files, share them with approved users, and download files from their friends list. FLoSS is the final project from Penn State's Object-Oriented Software Design and Construction course (SWENG311).


Rolly Cubed

Rolly Cubed is a physics puzzle game that requires you, the cube, to roll around the environment and complete a series of tasks. There are three game modes: Time Trial (race to the finish), Adventure (complete obstacle course), and Demolition (knock objects off of a platform). Rolly Cube was made using Unity as the final project for Introduction to Technical Game Development (GAME250). The game was an introduction to a 3D game engine, physics systems, models and textures, particle systems, and basic game development.
